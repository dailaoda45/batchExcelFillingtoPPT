# 批量生成 PPTX

根据 `data.xlsx` 中的数据，逐行替换 `模版.pptx` 中的占位符文字，自动计算合适的字体大小（支持多行换行），每行生成一份独立的 PPTX 文件。

---

## 目录结构

```
├── _run.py                主脚本
├── 模版.pptx               PPTX 模版（第一页作为模板）
├── data.xlsx               数据源
├── _生成结果/              输出目录（自动创建）
└── README.md               本文件
```

## 数据格式

`data.xlsx` 必须包含 **3 列**，列名任意，数据从第 2 行开始：
我的示例内容
| 列 | 内容 | 示例 |
|----|------|------|
| A  | 文件夹名称（输出文件名） | `一个世纪儿的忏悔-缪塞` |
| B  | 标题（替换【文档关键字[标题]】） | `一个世纪儿的忏悔` |
| C  | 副标题（替换【文档关键字[副标题]】） | `-缪塞` |

## 模版准备

1. 在 `模版.pptx` 的第一页放置两个文本框
2. 标题文本框写入 `【文档关键字[标题]】`
3. 副标题文本框写入 `【文档关键字[副标题]】`
4. 在 PowerPoint 中设置好字体、字号、颜色、文本框自动调整等样式

> 脚本通过搜索文本内容来识别哪个是标题、哪个是副标题，所以占位符中必须包含"标题"或"副标题"字样。如果有其他需要自行修改。

## 使用方式

### 方式一：直接运行 Python 脚本

```bash
# 安装依赖（仅首次）
pip install openpyxl lxml fonttools Pillow

# 修改配置后运行
python _run.py
```

### 方式二：使用打包好的 exe

把 `批量生成PPTX.exe` 放在 `模版.pptx` 和 `data.xlsx` 同级目录，双击运行即可。

打开 `_run.py`，找到第 35 行：


## 输出说明

- 每行数据生成一个 `.pptx` 文件，存放在 `_生成结果/` 目录
- 文件名使用 data.xlsx 第 A 列的值命名
- 每个文件只有 1 页，样式与模版第一页完全一致
- 标题字体大小会根据文本长度和文本框尺寸自动计算（代码模拟 PowerPoint AutoFit）

## 注意事项

1. **字体**：脚本使用系统字体「微软雅黑」进行文字宽度测量，确保 Windows 系统有此字体
2. **文件名**：输出文件名取自 data.xlsx 的 A 列，会自动过滤 `\ / : * ? " < > |` 等非法字符
3. **文本框识别**：脚本通过文本内容中的"标题"和"副标题"关键词来定位文本框，请确保占位符包含这些词
4. **临时文件**：运行时会在 `_tmp_fonts/` 生成临时字体文件，程序退出后自动清理
5. **大数量处理**：828 条数据大约需要 2~3 分钟，请耐心等待

## 依赖库

| 库 | 用途 |
|----|------|
| openpyxl | 读取 Excel 数据 |
| lxml | 解析/修改 PPTX 内部 XML |
| fonttools | 从 TTC 字体文件中提取 TTF |
| Pillow | 测量文字宽度（用于 AutoFit 计算） |
